# report.md — run 리포트 (runs/<env>-<run-id>/report.md)

> 매 run의 산출물이자 **비전공자가 읽는 첫 문서**. 유인(P3)·무인(P5) 공통 형식 —
> 무인 run에서는 러너가 생성하고, 사람은 통보 정책(qa-config.yaml `ops.notify.ack_hours`) 내
> 확인 책임이 있다. 증거 인용 전 redaction 완료가 전제다(원칙 6, patterns/evidence.md).
> 실패 3분류의 확정과 개선 실행은 retro.template.md(P4)로 이어진다 — 여기서는 잠정 분류까지만.
