# gate-summary — 게이트 요약 (4게이트 공통 고정 형식)

> 게이트에서는 이 형식으로 제시하고 **반드시 멈춘다**(AGENT.md §4, 원칙 10). 채팅 제시가 기본,
> 파일 저장은 선택 — 승인 결과는 STATE.md 결정 로그(append-only)에 즉시 기록한다.
> 비전공자가 읽는다: YAML·코드 원문이 아니라 평문으로 쓰고, 원문은 경로·링크로 첨부한다.
> 제시 순서 고정: **공통 4항목 → 해당 게이트 추가 항목 → 하단 응답 3어휘**. 항목 생략 금지 —
> 사용자가 "알아서 해"라고 해도 요약을 짧게 할 수 있을 뿐 게이트를 건너뛸 수는 없다.

## 게이트별 추가 항목 정의

### [게이트1] 설정 승인 — config 평문 번역 + 준비물 대장

<!-- qa-config.yaml 원문을 붙이지 말 것 — 평문 번역 표로 제시하고 원문은 경로 첨부.
     번역 행은 최소: 대상 환경 / 데이터 경계·정리 / 불가침(deny-list) / 예산 / 범위·상한 / 운영·통보. -->

| 설정 | 평문 번역 | 원문 필드 (qa/qa-config.yaml) |
|---|---|---|
| 대상 환경 | `<"stage(https://…)에서만 테스트합니다. prod는 대상이 아닙니다.">` | environments |
| 데이터 경계 | `<"qa- 로 시작하는 데이터만 만들고, 끝나면 지운 뒤 0건을 확인합니다.">` | data.test_prefix / retention |
| 불가침 목록 | `<"다음 n건은 기계적으로 차단됩니다: …">` | data.deny_list / risk.irreversible_actions |
| 예산 | `<"한 번의 실행에서 외부 호출 n회·생성 n건을 넘으면 스스로 멈춥니다.">` | risk.budget_per_run |
| 범위 | `<"우선 기능 n개, 케이스는 최대 n개까지만 만듭니다.">` | scope |
| 운영·통보 | `<"결과는 파일로 남기고, n시간 내 확인을 부탁드립니다.">` | ops.notify |

**준비물 대장(qa/clearance.md) 현황**: 확보 `<n>/<m>` — 미확보: `<CL-XX: 요지, …>`

### [게이트4] 스케줄 허가 — 3문항 고지 + 통보 확정

| 스케줄 | 값 |
|---|---|
| 주기 / 시간창 | `<매일 1회 / 02:00–04:00>` — 주기 > run 최대 소요 `<mm분>` 검증: `<통과/실패>` |
| 환경 | `<환경 이름>` (`unattended_allowed: true` 확인) |
| 기동 방식 | `<cron / OS 작업 스케줄러 / 에이전트 스케줄 기능>` + `<에이전트 회고 병행 여부>` |
| 중단 조건 | `<연속 n회 실패 시 중단+통보 — qa-config ops.failure_policy>` |
| 무인 범위 | 러너 편입 케이스 `<n>`건만 (미편입 `<m>`건은 유인 런에 남음) |

**3문항 고지 (그대로 인쇄)**

1. **무엇이 자동 실행되나**: `<러너 편입 케이스 n건 — 대상 기능·환경·주기를 평문으로>`
2. **최악의 경우 무엇이 잘못될 수 있나**: `<allowlist·예산 상한이 막아주는 것과 그래도 남는 위험을 정직하게>`
3. **멈추는 법**: `qa/.stop` 파일을 만들면 케이스 경계에서 멈춘다 — 원클릭 STOP: `<더블클릭 스크립트 경로>` (README 기재)

**통보 확정**: 채널 `<file(기본) / 푸시 채널>` · 확인 책임 `<n>`시간 내 (qa-config ops.notify) — `<사용자와 확정 완료 여부>`

## 응답 (아래 3개 중 하나로만 — 요약 하단에 항상 그대로 인쇄)

> - **"승인"** — 이대로 진행합니다.
> - **"수정: <내용>"** — 반영한 뒤 이 요약을 다시 제시합니다.
> - **"보류"** — 진행하지 않고 멈춥니다(STATE.md 기록, 재개 시 이 게이트부터).
