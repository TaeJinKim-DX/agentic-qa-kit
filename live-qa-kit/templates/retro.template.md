# retro.md — run 회고 (runs/<env>-<run-id>/retro.md)

> P4 산출물(phases/P4-retro.md). 같은 run의 report.md가 "무슨 일이 있었나"라면 이 문서는
> "그래서 무엇을 고치나"다. 실패를 3분류로 확정하고, 오탐 3관문(원칙 8)을 기록하고,
> 안정 케이스를 runner/에 편입한다. 정책 변경(케이스 증감·known-issue 등록·기준 변경·config 수정)은
> 여기서 실행하지 않는다 — 게이트 상정만(원칙 10). 기술 수정(셀렉터·대기·스크립트)은 직접 실행.
